## 🔴 P0 — 必做项（决定生死线）

### 1. 📱 微信小程序 C 端下单

> 答辩现场评委微信扫码 → 小程序内一键下单 → Web 大屏立刻弹出审批流 → 批准后无人机起飞。10 分钟内打通"微信下单→调度审批→3D 飞行"全闭环。
> 微信生态内扫码即开、无需输入网址、体验原生、答辩现场更具产品完成度的冲击力。

- [ ] 使用 **uni-app**（或原生微信小程序框架）开发极简物流下单小程序
  - 页面 1：首页 → 选择城市 + 地图点选起终点（可用腾讯地图 SDK）
  - 页面 2：确认下单 → 调用后端 `POST /api/tasks` 提交物流需求
  - 页面 3：订单追踪 → 实时展示订单状态流转（WebSocket/轮询状态）
- [ ] 后端 CORS 配置适配小程序请求域名（需要 HTTPS）
- [ ] 生成小程序码，答辩时投屏展示供评委扫码
- [ ] Web 大屏端已有 SSE 实时推送 → 零延迟联动，无需新增后端工作

**评估工作量**：小程序端约 3 个核心页面，复用全部现有后端 API

---

### 2. 🧠 AI 航线风险预审 — "AI+ 赛事必考题"

> **战略意义**：大赛通知第四条反复强调"AI+"，评审焦点在于"驾驭 AI 工具解决实际问题的策略"。接入国产 AI 工具（DeepSeek/通义千问）做智能预审，既满足赛事要求又构建技术纵深。

- [ ] **AI 预审 API** (`POST /api/ai/preflight-check`)
  - 输入：起终点坐标、载重、当前天气/风速
  - 调用：大赛指定的国产 AI 工具（DeepSeek / 通义千问任选其一）
  - 返回：风险等级（低/中/高）+ 自然语言解释 + 建议
  - 示例："当前风速 12m/s，该航线中段经过高楼密集区南山科技园，预计颠簸风险较高，建议提升巡航高度至 120m 或推迟 2 小时起飞。"
- [ ] 前端在提交任务对话框中集成"AI 预审"按钮
  - 1 秒内返回结果，以卡片形式呈现风险评估
  - 高风险时自动标红并建议驳回
- [ ] 创建 `/api/ai/chat` 端点 — 自然语言问答调度台
  - 示例："深圳南山区今天飞了多少架次？" → AI 查询 FlightLog 返回答案
  - 从数据库检索、生成回答，演示"AI + 数据库"的协同能力

**AI 工具合规性**：使用大赛附件 4 指定的 DeepSeek 或阿里通义系列

**评估工作量**：后端新增 2 个 API + 前端 1 个对话组件，**约 1.5 天**

### 4. 🔴 4D 空域冲突检测与预警

> 体现"管控平台"核心价值——同时段多架无人机的时空碰撞预判

- [ ] 在 `animation.ts` 的每帧更新中，检测所有活跃 UAV 之间的距离
  - 两架 UAV 水平距离 < 200m 且高度差 < 30m → 触发黄色预警

- [ ] 大屏渲染：冲突 UAV 之间显示红色连接线 + 闪烁标记

- [ ] 后端：调度时 AI 自动推荐"延迟 60s 起飞"或"航高 +20m"
