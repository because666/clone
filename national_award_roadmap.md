#### 1. 用户体系与权限管理
- 实现登录/注册功能（JWT Token 认证）
- 角色分级：**管理员**（全权）、**调度员**（航线管理）、**查看者**（只读大屏）
- 操作日志记录与审计
- **加分点**：这是 "Web 应用" 赛道区别于纯可视化的关键分水岭

#### 2. 数据库持久化
- 引入 PostgreSQL + PostGIS（空间数据库，天然契合地理场景）
- 核心数据表：用户表、航线任务表、无人机注册表、飞行日志表、告警记录表
- 把当前预生成的静态 JSON 迁移到数据库，实现真正的 CRUD
- **加分点**：PostGIS 空间查询能力（如"查询某区域 500m 内的所有航线"）

#### 3. 航线任务管理工作流
- **任务创建**：从前端交互式选择起终点 → 后端生成轨迹 → 提交审批
- **审批流程**：审批中 → 已通过 → 执行中 → 已完成（状态机）
- **任务列表页**：可搜索、可筛选、可排序的任务管理表格
- **加分点**：体现了系统的"业务闭环"能力

### 🟡 P1 — 技术深度提升项

#### 4. 数据统计分析大屏（第二屏幕）
- 新增一个独立的**数据分析仪表板页面**（与 3D 地图页并列）
- 内容包括：
  - 📊 **各城市航班量趋势图**（折线图/柱状图）
  - 🔥 **起降点热力分布图**（Heatmap）
  - 📈 **能耗统计直方图**（按载重/距离/天气分组）
  - ⏱️ **航班时段分布**（每小时航班密度）
  - 🚨 **告警统计**（电量告警、禁飞区告警的占比与趋势）
- 推荐用 **ECharts** 或 **Recharts** 来做传统二维图表
- **加分点**：评审最爱看"一目了然"的数据故事

#### 5. AI 智能化增强（赛事鼓励 AI+）
- 基于历史轨迹数据，训练一个**航线能耗预测模型**（简单线性回归/随机森林即可）
- 输入：距离、载重、天气、风速 → 输出：预测耗电百分比
- 在创建任务前提供"**AI 预审**"：预测该航线能否安全抵达
- 或者：基于时段/天气/历史数据，预测某个区域的**空域拥堵指数**
- **加分点**：满足赛事 "AI+" 的核心评审导向，使用大赛指定的国产 AI 工具辅助开发

#### 6. 多无人机冲突检测与协同调度
- 同一时段多架无人机的**4D 航路冲突检测**（时间+空间的碰撞预判）
- 可视化展示冲突点（红色闪烁标记）
- 简易调度建议：自动推荐延迟起飞或改变航高
- **加分点**：体现"管控平台"的核心价值
